About This Site

Technologies

This was built with the following technologies:

  • Sass: because variables and mixins make life fun.
  • RequireJS: used because it's awesome for dependency management writing object oriented Javascript
  • Handlebars: Makes templating prettier and more readable.
  • Grunt: using a build step is always awesome and you can use Grunt Includes for front-end templating
  • Bower: package management is super sleek.

CSS Architecture

I'm a big fan of a combination os OOCSS and SMACSS architeture styles. Files are divided between landmarks, container objects, content, layout directories.

  • Landmarks: occur consistently across every page.
  • Content: styles content, such as text or inputs
  • Container: act as wrappers for content, usually for providing vertical rhythm
  • Layout: provide overall structure to a page or elements with a specific purpose.

JS Architecture

Javascript files use Require to handle inter-file dependency and are structured in the following directories that correspond with their purpose:

  • Controllers: overall controller for the app. Only one since this is a single page app and the no need for multiple controllers.
  • Views: responsible for rendering the content onto the page
  • Services: have a specific purpose, such as retrieving data from a server in this case. Services can be anything that you might want to call from any controller. Similar to Angular, in a way.
  • Models: structure the data for use. Usually models only need to be called froma service which retrieves data, but they're not limited to that. Even if data is simple, using a model to strucutre the data ensures that any server side changes only require a change of one file and no others.
